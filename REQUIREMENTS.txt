 SECTION 1 — CLARIFIED 需求 (REQUIREMENTS)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PRIMARY USE CASE
  - Vendor at a busy art fair table, phone in hand
  - Customer picks multiple items; record the whole interaction in under 5 taps
  - Check totals at end of day / for tax/accounting purposes
  - Everything backed up to Google Sheets automatically

CORE FEATURES

  [1] ITEM CATALOG
      - List of items with name, price, and optional stock count
      - Items are pre-configured (not created during the fair)
      - Can have category grouping (e.g. Zines, Prints, Books, Stickers)
      - Each item has a thumbnail color or emoji for quick visual ID
      - Admin mode to add/edit/remove items and prices and edit stock count

  [2] SALE RECORDING (interaction-based)
      - A sale = one customer interaction, which may include multiple items
      - Tap item cards to add them to the current cart (tap again to increment)
      - Cart summary bar always visible at bottom of Items view:
          "[n items · $XX.XX]  [  CHECKOUT  ]"
      - Tap CHECKOUT → Checkout modal appears with:
          - List of items in cart (name, qty, line total) — editable (qty +/− per line, remove line)
          - Payment method selector: Venmo / Zelle / Cash
          - Optional note field (e.g. "gave discount", "trade")
          - Total for this transaction
          - [Cancel]  [✓ CONFIRM SALE]
      - On confirm: transaction is logged instantly with timestamp
      - Undo last transaction (within 30 seconds): reverses the entire transaction
          — all line items removed, full amount deducted from totals, stock restored
      - Stock deduction per item on confirm

  [3] PAYMENT METHOD TRACKING
      - Three methods: Venmo | Zelle | Cash
      - Live running totals per method visible at all times
      - Grand total displayed prominently
      - End-of-day breakdown (how much cash to count, how much digital)

  [4] GOOGLE SHEETS SYNC
      - Each confirmed transaction → multiple rows in Sheets (one row per line item),
        all sharing the same transaction_id
      - Offline queue: if no internet, transactions are queued and synced when back online
      - Sync status indicator (green = synced, yellow = pending, red = error)
      - Sheet also stores item catalog (separate tab) for cross-device consistency

  [5] DASHBOARD / SUMMARY
      - Today's total revenue
      - Breakdown: Venmo / Zelle / Cash totals
      - Best-selling items (by count and by revenue)
      - Sales log (most recent first, scrollable) — one row per transaction,
        expandable to see individual line items
      - Can filter log by payment method

NICE-TO-HAVE (Phase 2)
  - Multi-day event support (date filter)
  - Export/print end-of-day report
  - QR code display for Venmo/Zelle (customer scans to pay)

  ─────────────────────────────────────────────────────────────────────────
  INTERACTION SPECS
  ─────────────────────────────────────────────────────────────────────────

  ITEM CARD (grid items — Zine A, Zine B, Print, Sticker)
  ────────────────────────────────────────────────────────
  Idle:
    - dark card, emoji + name + price + stock count
    - stock: dim gray label; ≤3 left → amber; 0 → "sold out", card dims to 35% opacity
    - min touch target: full card face (≥44×44px)

  Tap (qty = 0 in cart):
    - press: scale 0.94, 80ms ease-in
    - release: scale 1.0, 150ms ease-out
    - badge ×1 slides in from top-right (scale 0→1, 200ms spring)
    - card border flashes accent green, 500ms fade out
    - [−] button fades in bottom-left of card, 200ms
    - cart bar slides up if first item, 250ms ease-out

  Tap (qty > 0 in cart, tap card body):
    - same scale animation
    - badge number pops: scale 1.3→1.0, 150ms spring

  Tap [−] button (bottom-left of card):
    - button: scale 0.86→1.0, 100ms spring
    - badge decrements
    - at qty = 1→0: badge scale 1→0 (200ms), [−] fades out (200ms)
    - if cart becomes empty: cart bar slides down, 200ms ease-in

  Long-press (600ms):
    - at 400ms: card border brightens (early feedback)
    - at 600ms: subtle haptic, card scale 1.02
    - Edit Item sheet slides up

  BOOK ROW (full-width list row, always below the card grid)
  ──────────────────────────────────────────────────────────
  Idle (not in cart):
    📚  Book          5 left          $40.00  [+]
    - [+] is a 44px tap target on the right
    - tap anywhere on row OR tap [+]: qty +1
    - [+] scales 0.86→1.0, 100ms spring; row bg flashes green 400ms

  In cart (qty > 0):
    📚  Book          5 left    [−]  ×1  [+]  $40.00
    - [−] and [+] appear inline via fade+slide, 200ms
    - qty number pops on change: scale 1.3→1.0, 150ms

  Long-press row: same as card — edit sheet slides up

  STOCK SETUP SHEET  (tap [📦 Stock] in header)
  ──────────────────────────────────────────────
  - Sheet slides up, 250ms ease-out
  - Lists all items: emoji + name | [−] | [qty field] | [+]
  - Qty field: tap to focus → numeric keyboard opens, pre-selected text
  - [−][+]: fine-tune without keyboard
  - Current stock pre-filled; blank if never set
  - [Done] saves all and closes sheet
  - Designed for: head-down, 5 items, under 30 seconds

  CHECKOUT MODAL
  ──────────────
  - Slides up 250ms ease-out; backdrop fades in simultaneously
  - Line item qty [−]: disabled + dimmed at qty=1 (use [×] to remove)
  - Line item [×] remove: row slides left + fades, 180ms ease-in
  - Payment pill: tap fills pill with method color, scale 1.05, 140ms
  - SOLD button: greyed + disabled until payment selected
  - SOLD tap: spinner 300ms → success checkmark flash → modal slides down 300ms

  UNDO TOAST
  ──────────
  - Slides up above nav, 280ms ease-out
  - Sits above cart bar if cart bar is also visible
  - Timer shows seconds counting down (30→0)
  - UNDO: toast slides down immediately, all state reversed
  - Auto-dismiss at 0: slides down

  SYNC INDICATOR (top-right of header)
  ─────────────────────────────────────
  - Synced:  solid green dot · "synced"
  - Syncing: amber pulsing dot · "syncing…"
  - Error:   red dot · "sync failed" — tap to retry
  - Offline: grey dot · "offline"
  - Color crossfade: 300ms

  REDUCED MOTION
  ──────────────
  - All transforms and fades disabled if prefers-reduced-motion: reduce
  - State changes still visible (color, opacity) but instant

 SECTION 4 — INTERACTION SPECS (Interaction Designer)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ITEM CARD (tap to add to cart)
  Hover/Touch:
    - Scale: 1.03, shadow deepens
    - Transition: 150ms ease-out
    - Cursor: pointer
  Active (press):
    - Scale: 0.96
    - Transition: 80ms ease-in
  Added to cart flash:
    - Background pulses green for 600ms
    - If already in cart, show a qty badge (e.g. "×2") on the card
    - Easing: ease-out fade back to normal
    - Non-blocking (card is immediately tappable again)

CART BAR (bottom of Items view)
  - Hidden when cart is empty (slides up into view on first item added)
  - Shows: "[n items · $XX.XX]  [ CHECKOUT ]"
  - Updates instantly on each tap
  - Checkout button: accent color, full-width feel

CHECKOUT MODAL
  Enter: slide-up from bottom + backdrop fade-in, 250ms ease-out
  Exit:  slide-down + backdrop fade-out, 200ms ease-in
  Line item qty [+/−] buttons:
    - Tap: scale 0.9 → 1.0, 100ms spring
    - Disabled at 1 (can remove line instead) or max (opacity 0.3)
  Payment pill selection:
    - Selected: background fills, scale 1.05, 150ms ease-out
    - Unselected: border only
  SOLD button:
    - Idle: accent color (#4caf85 or similar)
    - Hover: brighten 10%, scale 1.02
    - Loading: spinner, disabled, opacity 0.7
    - Success: checkmark flash, modal auto-closes after 300ms

UNDO TOAST
  - Slides in from bottom after transaction confirmed
  - Countdown ring animation (30s timer)
  - Tap UNDO: reverses entire last transaction (all line items removed,
    stock restored, totals updated)
  - Auto-dismiss after 30s with slide-out animation

SYNC STATUS INDICATOR (top-right)
  - Synced: green dot, "synced" label
  - Pending: yellow pulsing dot, "syncing..."
  - Error: red dot, "sync failed" — tap for retry
  - Transition: 300ms color crossfade

BOTTOM NAV
  - Active tab: accent underline + label bold
  - Tap: slight scale 0.95 → 1.0 on icon, 100ms
  - Tab switch: content crossfades 150ms ease-in-out

SUMMARY TOTALS
  - On mount or update: numbers count up from 0 to final value, 400ms ease-out
  - Payment bars: width animates from 0%, 500ms ease-out
  - Staggered: each bar delays 80ms after previous

REDUCED MOTION
  - All animations disabled if prefers-reduced-motion: reduce
  - State changes remain instant but visible

 SECTION 5 — GOOGLE SHEETS SCHEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TAB: "Sales"
  One row per line item. Rows from the same transaction share a transaction_id.
  Columns: transaction_id | timestamp | item_name | quantity | unit_price | line_total
           | payment_method | note | synced_at

  Example (one 3-item transaction):
  txn_001 | 2:34pm | Zine A  | 1 | 8.00  |  8.00 | Cash | ""  | 2:34pm
  txn_001 | 2:34pm | Print   | 1 | 25.00 | 25.00 | Cash | ""  | 2:34pm
  txn_001 | 2:34pm | Sticker | 2 |  3.00 |  6.00 | Cash | ""  | 2:34pm

  Note: payment_method and note are the same for all rows in a transaction.
  transaction_id format: e.g. txn_YYYYMMDD_HHMMSS or a short UUID.

TAB: "Items"
  Columns: id | name | emoji | price | category | active

TAB: "Summary" (auto-generated by Apps Script formula, read-only)
  Shows: total revenue, breakdown by method, top items, transaction count
